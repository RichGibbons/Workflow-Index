Introduction

Workflow is the process which items of work go through in order to be recorded and moved around on their journey to being complete. Each item of work is prioritised, routed and the obtained by a member of staff to be processed.

Supporting prototypes

Production Information (May)

The purpose of this user research is to Usability test the Production Information prototype to establish how readily users can access and navigate to the Production Information available.

'Consider Assignment from' (April)

The purpose of this prototype is to establish whether the proposed language for ‘Next Action’ where a caseworker is assigned and receives work using the Manage Assigned function meets user needs.

This is to bring the language used for referrals and assigned work in line with each other.

The proposed Next Action language is ‘Consider assignment from ’

Manage Assigned Work (February)

The purpose of this research was to establish meaningful language to use when displaying an error message in the scenario where a workflow application was open in CORA, and it was ‘manage assigned’ to another user.

The user the attempts to Refer or Defer the application .

Select a category of work - ERU (January)

Executive Register Updates will introduce four new categories of work.

The purpose of this prototype was to explore the usability of the components for choosing a category of work with users using two prototypes.

In particular we wanted to focus on the time taken and ease of completion of the task of selection of categories using both prototypes to ascertain which is more efficient and meets user needs.

Refer

  • Refer - List A2 (January)
  • Refer - List A1 (January)
  • Refer - Tabs (January)
  • Refer - Tabs (December)

The purpose of this research is to:

  • establish whether the refer prototype meets the user needs for a minimum viable product.
  • confirm whether users need notification messages when they receive a referred application.
  • identify any potential issues with the User journey.
  • establish whether users can complete their normal user journeys using the prototype.
  • establish what user expect the Date obtained to be on receiving a referred application.
  • establish whether users need to see the name of the person they have referred an application to in the confirmation of action message.
